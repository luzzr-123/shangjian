# PRD - NoteFlow 安卓个人多功能记事软件

## 1. 文档信息
- 产品名称：NoteFlow
- 版本阶段：V1.0 MVP
- 平台：Android 原生
- 目标用户：个人用户，偏重自我管理、待办执行、习惯养成、结构化记录
- 产品目标：在一个本地优先的软件内，完成“任务 + 习惯 + 笔记”的统一管理，并提供细粒度提醒控制能力

## 2. 产品定位
NoteFlow 是一款面向个人的多功能记事软件，核心特征不是“功能多”，而是：
1. **任务、习惯、笔记统一管理**
2. **提醒控制粒度高**
3. **支持子项拆解**
4. **本地优先，稳定可靠**
5. **界面极简、安静、低压迫感**

## 3. 目标用户
### 核心用户画像
- 需要管理日常任务的人
- 希望建立长期习惯的人
- 有碎片记录与结构化笔记需求的人
- 对提醒频率、免打扰时间、执行窗口有较高要求的人

### 不作为首版重点的人群
- 团队协作用户
- 多端实时同步强需求用户
- 需要复杂知识图谱功能的用户

## 4. 核心问题
当前常见软件往往存在这些问题：
1. 任务、习惯、笔记分散在不同应用
2. 提醒机制粗糙，仅支持单次通知或简单重复
3. 子任务与时间窗口规则不灵活
4. 笔记与任务详情编辑能力割裂
5. 过度依赖云端，离线体验不稳定

NoteFlow 要解决的是：  
**把“记录、提醒、执行、复盘”放在一个本地优先的个人工具中。**

## 5. 产品目标
### V1.0 必达目标
- 完成任务、习惯、笔记三模块
- 支持子任务 / 步骤
- 支持开始提醒、特别提醒、重复提醒、截止时间
- 支持免打扰时间
- 支持 Markdown + 图片插入
- 支持编辑、删除、回收站
- 支持本地备份恢复

### V1.0 不做
- 云同步
- 团队共享
- OCR
- AI 自动分类
- 桌面端 / iOS 端
- Web 端

## 7. 详细需求
### 7.1 首页 / 今日总览
目标：用户打开应用时，快速知道今天该做什么。

展示内容：
- 今日待办数量
- 今日习惯待执行数量
- 今日已完成数量
- 最近编辑的笔记
- 快速新建入口

交互：
- 可直接完成任务 / 子任务
- 可直接完成习惯打卡
- 可跳转详情页
- 空状态下提供显眼的新建入口

### 7.2 任务模块需求
字段：
- 标题（必填）
- 详情（选填，支持 Markdown）
- 标签（选填）
- 优先级（普通 / 高 / 紧急）
- 子任务列表（选填）
- 开始提醒时间（选填）
- 提醒窗口结束时间（选填）
- 截止时间（选填）
- 重复提醒间隔（选填）
- 特别提醒时间列表（选填）
- 全天模式（布尔）
- 完成规则（手动完成 / 全子任务完成自动完成）

列表展示：
- 标题
- 剩余时间 / 截止状态
- 子任务完成进度
- 紧急状态
- 下一次提醒时间（可选）

排序逻辑（默认）：
1. 紧急任务
2. 已逾期任务
3. 今天到期
4. 即将到期
5. 其他任务

允许的操作：
- 新建
- 编辑
- 删除
- 恢复
- 标记完成
- 取消完成
- 完成子任务
- 调整优先级
- 添加 / 删除子任务

提醒逻辑：
- 到开始提醒时间后进入提醒窗口
- 在提醒窗口内，未完成则按重复间隔提醒
- 特别提醒时间优先触发
- 免打扰时间内不提醒
- 完成后取消后续提醒
- 超过截止时间后标记逾期

### 7.3 习惯模块需求
字段：
- 标题（必填）
- 详情（选填）
- 频率规则（必填）
- 执行窗口开始时间（选填）
- 执行窗口结束时间（选填）
- 重复提醒间隔（选填）
- 特别提醒时间列表（选填）
- 步骤列表（选填）
- 打卡模式（勾选 / 时长）
- 目标时长（时间型时使用）

频率规则：
- 每日
- 每周指定几天
- 每隔 N 天
- 每月指定几天

完成逻辑：
- 勾选型：手动打卡即完成
- 步骤型：全部步骤完成即完成
- 时长型：达到目标时长才完成

展示内容：
- 今日是否需要执行
- 当前周期剩余时间
- 连续天数
- 今日是否已完成

### 7.4 笔记模块需求
字段：
- 标题（必填）
- 正文（支持 Markdown）
- 图片附件（支持正文中插入）
- 标签（选填）
- 创建时间
- 更新时间

操作：
- 新建
- 编辑
- 删除
- 恢复
- 搜索
- 查看图片大图

编辑体验要求：
- 自动保存草稿
- 支持常用 Markdown 语法
- 支持在正文中间插入图片占位并渲染
- 支持基础列表与勾选列表

## 8. 免打扰规则
用户可设置全局免打扰时间段，例如：
- 12:00 - 13:00
- 23:00 - 07:00

规则：
- 在免打扰时间内不弹提醒
- 不进行集中补发轰炸
- 结束后按下一次正常提醒节奏继续

## 9. 编辑、删除、回收站、恢复
编辑：
- 所有任务 / 习惯 / 笔记均可再次编辑
- 已完成任务允许恢复为未完成
- 已打卡习惯允许撤销本次记录

删除：
- 首版默认软删除
- 删除内容进入回收站
- 支持手动清空
- 默认可保留 30 天（实现上可不强制自动清理）

## 10. 备份与恢复
导出：
- 导出 JSON 数据
- 导出媒体文件目录
- 打包为 zip

导入：
- 读取 zip
- 恢复结构化数据
- 恢复图片映射关系
- 对冲突项给出提示（首版可简单处理）

## 11. UI 与视觉要求
风格关键词：
- 极简
- 柔和
- 安静
- 低饱和
- 圆角
- 大留白
- 卡片式布局
-导航栏半隐藏设计，导航栏背景与界面背景融为一体。

主题分色建议：
- 任务：淡紫蓝
- 习惯：薄荷绿
- 笔记：米黄色

关键原则：
- 可读性高于纯视觉效果
- 激活态不能只靠发光，必须有底色与文本对比变化
- 首页要有信息密度，不能只有空白感

## 12. 非功能需求
- 本地优先
- 启动速度快
- 离线可用
- 提醒稳定
- 数据不易丢失

# UI Style Guide - NoteFlow

## 1. 风格目标
视觉方向参考：
- 极简卡片式布局
- 柔和背景
- 低饱和色块
- 大圆角
- 留白充足
- 微弱发光与阴影

但需要升级为：
- 更高文字对比度
- 更强信息层级
- 更可读的列表与表单

## 2. 配色建议
### 基础背景
- 主背景：暖白 / 极浅灰
- 次级背景：更浅层卡片白

### 模块色
- 任务：柔和淡紫蓝
- 习惯：柔和薄荷绿
- 笔记：柔和米黄

### 状态色
- 紧急：偏暖红但低饱和
- 逾期：偏红
- 完成：柔和绿色
- 提醒：蓝紫强调

## 3. 字体层级
- 页面标题：较大、半粗
- 卡片标题：中等偏大、半粗
- 次级信息：常规字重
- 说明文：偏浅但必须保证对比度

不要为了“高级感”把正文做得太灰。

## 4. 组件规范
### 4.1 卡片
- 大圆角
- 轻微阴影
- 内边距充足
- 信息不要过度拥挤

### 4.2 按钮
- 主按钮：有底色
- 次按钮：描边或浅底
- 危险动作：明确区分

### 4.3 Chips / 标签
- 激活态：底色 + 文本对比变化
- 不要只靠发光

### 4.4 浮动按钮
- 首页与列表页可有统一新建按钮
- 不要过大，避免压迫内容

## 5. 页面风格重点
### 今日页
要有“总览感”，不能过空。
建议包含：
- 顶部日期
- 今日摘要
- 今日任务
- 今日习惯
- 最近笔记

### 列表页
要兼顾美观与扫描效率。
建议：
- 卡片高度统一
- 重要信息置顶
- 次要信息折叠

### 编辑页
必须重视填写体验：
- 字段间距大
- 时间设置区域清晰
- 子任务 / 步骤编辑方便
- 图片插入入口清晰

## 6. 动效建议
- 切换 Tab：轻量过渡
- 卡片完成：柔和反馈
- 新建页打开：平滑进入
- 不要使用夸张动画

## 7. 可用性优先级
视觉美感 < 可读性 < 交互效率 < 稳定性
